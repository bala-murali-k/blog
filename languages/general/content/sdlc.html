What is software development life cycle?

The software development is a process of planning, writing, modifying and maintaining software. The main goal of SDLC is to minimize the risks long before deciding how to launch the product and ensure the project meets the customer's expectations on both during and after the production.

Note:

The term "life cycle" was first used in 1950's and 1960's for the stages involved in developing a new computer system. Now it is commonly used to refer all stages in the production of any type of software.

what is the benefits of SDLC?

  1. Increased visiblity of the development process for all stake holders involved.
  2. Understanding the requirements and the goal of the software.
  3. Identify risks at early stage to improve risk management and cost estimation.
  4. A systemetic approach to delivering software that meets customer satisfaction such as building prototypes or writing functional specifications.
  5. Track the progress on goals.

Generally there is seven stages in software development life cycle. Each stage has it's own set of activities that the team members need to do, where the timeline can be vary from project to project but generally follow these phase. The SDLC process can be a little different for every team and product. Let's look at the seven phases.

Phase 1: Planning and Analysis

The first step in the software development life cycle is planning. It is where you gather buisness requirements from client and stake holders, set goals, identify risks and brainstorm ideas. Here the team will work together to devise a set of buisness goals, requirements, specifications, and any high level risks that might hinder the project success.

To properly decide what to make and what not to make, we can use a "feature prioritization framework" that takes into account of the software, the cost, the time it takes to build and other factors.

Phase 2: Analyze requirements

This phase is critical for converting the information gathered during the planning and analysis phase into clear requirements for the development team (tasks). This requires a lot of research and planning to create a detailed project plan document and work breakdown structure that outline the requirements ensuring that the project meets all the expectations.

This process guides the development of several important documents: SRS - software requirement specification, use case document and requirement traceability matrix document.

Phase 3: Design

The original plan and vision are eloborated into a software design document (SDD) which includes system design, programming language, templates, platform to use and application security measures. Here is also where we flowchart how the software behaves when there is an action.

The design phase will include the development of the prototype model, a pre-produced product that give the opportunity to visualise the project and change without the hazzle of rewritting the code.

Phase 4: Development

This is the phase where the actual coding takes place, the team members divide the project into software modules and turn the software requirements into code that makes the product.

This is the phase which consumes a lot of time and specialised development tools. It's important to have a set timeline and milestones so that the develoers can keep track of the process in this stage. The team will build the functionality for the product or service, which includes creating the user interface and building the database.

In some cases the development phase also merge with testing phase where certain tests are run to ensure that there is no critical bugs.
